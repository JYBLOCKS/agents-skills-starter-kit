# docs-sync

## Purpose

Keep README, guides, templates, and examples aligned with shipped behavior and structure.

## Supported Tools

- generic
- openai
- claude-code
- opencode

## Inputs

- changed structure or behavior
- existing docs

## Outputs

- updated docs plan
- drift notes
- sync checklist

## Guardrails

- Document current reality.
- Prefer concise onboarding paths.

## Process

1. Identify drift.
2. Update top-level docs first.
3. Sync templates and examples.

## Adapter Notes

- generic: update README when public structure changes.
