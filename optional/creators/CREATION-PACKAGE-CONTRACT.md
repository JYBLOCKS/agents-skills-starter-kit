# Creation Package Contract

Every creator must produce one final creation package with these sections:

## Required Sections

- Artifact Type
- Requested Outcome
- Normalized Name and Path
- Decisions Made
- Assumptions Taken
- Required Files to Create or Update
- Complete Draft Contents
- Validation Checklist
- Follow-Up Notes

## Output Expectations

- Draft contents should be ready to create or paste into files.
- Validation should reference the artifact's governing contract.
- Follow-up notes should only mention real dependencies, not generic advice.
